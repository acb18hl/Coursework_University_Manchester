Discussion of Q1.1a:¶

Overall, working with this dataset, the k-means algorithm shows better performance than single-linkage. Normally, choosing a suitable k value for the k-means algorithm is a challenge for users. It will affect the performance of k-means[1]. For this dataset, it provided labels in advance. This point makes the k-means algorithm perform well, and the average score reaches 0.8 in this case. However, there is another point worth noticing.From the output bar chart, it is clear that an error bar is shown in the k-means performance score. In the data visualization domain, an error bar is a helpful tool. It purposes to figure out any estimated error or uncertainty problem in the graph [2].

Such an estimated error in the k-means algorithm is a reasonable result. The working step of the k-means algorithm is[1]:

  1. Choose initial centroids randomly.
  2. Assigning each point to the closest centroid, then that is the initial clusters.
  3. For each cluster, get the average of all points to get new centroids. Then assign all points to the closest centroid. Then we can get the new cluster.
  4. Repeat step 3 to get a new centroid and new cluster until all points stop changing clusters.

Therefore, the unstable problem is caused by choosing the initial centroid. Since for every independent run, the algorithm chooses different initial centroid randomly. Tan states that poor initial centroids are possible to get poor results. One of the solutions to figure out this random problem is to run the k-means algorithm multiple times. It is able to reduce the effect of random initial centroid problems. Then the result could be more reliable .

Discussion of Q1.1b:¶

From the result of this dataset, the k-means algorithm shows significantly better performance than single-linkage. One of the points worth noticing for this output is the performance score of the single-linkage algorithm. The single-linkage algorithm is one kind of hierarchical clustering. The step and principles of the single-linkage algorithm are straightforward. Firstly, all elements are in a single cluster. Each cluster only contains one element. In each iteration, merge two closest clusters until all elements are in one cluster. The distance between two clusters depends on the elements from these two clusters [1]. The limitation of a single-linkage algorithm is noise and outliers. To clarify this problem. Assuming that is a case:

Cluster 1: element1(1,1), element2(1,2)

Cluster 2: element3(1,-1), element4(2,-2)

Cluster 3: element5(-2,1), element6(-2,2)

In this case, due to the distance between element1 and element3 are closest. Then Cluster 1 and Cluster 2 will be merged into one cluster. However, consider elements 4 and 6. The distance between these two elements is closer than between elements 2 and 4. Therefore, for each iteration, the two merged clusters may not be the closest clusters. This kind of case is a potential problem to lead single-linkage algorithm not accurate enough. Especially for data set containing noise data and outliers, the single-linkage algorithm is only able to focus on two elements with the shortest distance by each iteration. Any noise and outliers data will affect the iteration to cause poor results.

The discussion of q1.2a¶

For estimate the true number of clusters, there are two sensible approach called "Silhouette Coefficient" and "Calinski-Harabasz Index".

Silhouette Coefficient¶

The silhouette coefficient is a metric used to measure the performance of a clustering technique [3]. The result of the silhouette coefficient will between -1 to 1. The negative silhouette value indicates clusters assigned in the wrong way. 0 indicates the difference between each cluster is not clear enough. 1 indicates clusters with fair distance, and it is able to distinguish accurately.

Therefore, it could set a range of the true number of clusters. Then run the k-means algorithm with these values as k value. Storing these results to the data set. Finally, use silhouette coefficient to evaluate the set of k-mean algorithm results. The higher score indicates the corresponding number of clusters is suitable for the dataset.

The potential issue of the silhouette is the clusters algorithm may affect the result of the silhouette score. Normally, convex clusters will get a higher Silhouette score than other concepts of clusters [3]. To reduce the effect of issues, users could run different algorithms and compare the Silhouette score to get a better result.

Calinski-Harabasz Index¶

Another method called "Calinski-Harabasz Index", it is also called "Variance Ratio Criterion". The compute speed of this method is fast. This method will take all clusters, Then compute the ratio of between-clusters dispersion and inter-cluster dispersion for all clusters[3]. Higher scores indicate better performance. Therefore, it could set a range of values again. Then run the k-means algorithm with this set of ranges as k value. Using this method to evaluate the k-mean algorithm results. The highest Calinski-Harabasz means the corresponding number of clusters is suitable for the dataset, and it could be the true number of clusters for the data set.

The issue of Calinski-Harabasz is similar to Silhouette Coefficient. The convex clusters may get better Calinski-Harabasz score than other concepts of clusters.

Q2.1 (3 marks)¶

In Weka, run Apriori on this dataset using default settings. Comment on any patterns you see in the rules that are generated. Also discuss their support, confidence, and lift, demonstrating that you understand how they are calculated, their role, and how to interpret these values.

Q2.1 Answer¶

When run Apriori on this dataset with default setting, we need to focus on the support, confidence, and lift in the association rules. The association rules are able to find associations and correlation relationships for the large size of data. The support indicates the sum number of transactions, which include items in the antecedent and consequent parts of the rule [4].

The calculation of support of X is (transaction with event X)/(Total number of transaction)

Another key element is called confidence, it is the possibility of event B happening when event A has already happened [5]. The lift is the possibility of event B happening when event A has already happened. Furthermore, it is also consider the support of event B. It is the ratio of Confidence to Expected Confidence[4].

The Confidence(A->B) = Support(A,B)/Support(A)

At the same time, it is able to provide the information about the increase in the probability of consequent given the antecedent.

The lift(A->B) = Support(A,B)/Support(A)* Support(B)

Therefore, the Minimum support: 0.45 (196 instances) indicates the popular thershold has been set as 0.45. Only support larger than this thershold will be consider as important event. Minimum metric : 0.9 means only confidence larger than 0.9 will be seen as possible event. The lift:(value) for each best rule is used to measure the importance of a rule [6].
