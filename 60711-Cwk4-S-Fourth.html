Explain why it is not correct to build predictive models for "Treatment_Response" using records where it is missing?

Q1.2 Answer¶

The missing data is a challenging part to build predictive models. The missing value will potentially cause the inaccurate decisions in the modeling phase [1]. Therefore, when we build predictive models, it needs to do some pre-processing to handle missing values to avoid misleading predictions from the missing value [2]. Therefore, build predictive models with missing the "Treatment_Response" record is not a sensible approach. It increases the possibility of the model making the wrong decisions.

def get_data():

    data, labels = make_classification(
        n_features=2, n_redundant=0, n_informative=2,

    # Add some noise

    return data, labels

def plot_boundary(X, ax, clf):
    # Plotting decision regions
    x_min, x_max = X[:, 0].min() - 1, X[:, 0].max() + 1
    y_min, y_max = X[:, 1].min() - 1, X[:, 1].max() + 1
    xx, yy = np.meshgrid(
        np.arange(x_min, x_max, 0.1),
        np.arange(y_min, y_max, 0.1)
    )
    Z = clf.predict(np.c_[xx.ravel(), yy.ravel()])
    Z = Z.reshape(xx.shape)
    ax.contourf(xx, yy, Z, alpha=0.4)
    return ax
    
def boundary_full(data, labels, model, name, **kwargs):
    # Create estimator/model/classifier
    clf = model(**kwargs)
    # Fit the classifier
    clf.fit(data, labels)

    # Create axis
    fig, ax = plt.subplots()
    # Call the provided function
    ax = plot_boundary(data, ax, clf)
    # Now add the data (using scatter)
    # Ensure to colour the points according to the prediction
    ax.scatter(data[:,0], data[:,1], c=labels, s=20, edgecolor="k")
    # Format the graph...
    # Output as score.
    plt.title(name)
    plt.xlabel("feature 1")
    plt.ylabel("feature 2")
    print(clf.score(data, labels))

The performance of these three algorithm¶

The zeroR is the simplest classification algorithm [8]. It relies on the target and will ignore all predictors. It will create a table with frequency for the target and select its most frequent value. Therefore, this algorithm is only able to predict the majority category. It makes the decision boundaries does not worked in this algorithm. In our experiment, the accuracy of this algorithm is only 0.5. However, considering how this algorithm works, this accuracy is a reasonable value.

The Knn algorithm is a simple Machine Learning algorithms [9]. Firstly, this algorithm will select k number of neighbors. Then calculate the Euclidean distance of k number of neighbors. Thirdly, taking k nearest neighbors by Euclidean distance. Then in these k neighbors, count the number of the data in each category. Finally, assign new data to the category in which the number of the neighbor reaches the maximum value. The advantage of this algorithm is it is able to handle noisy data and performed well in large size of data [9].

Therefore, it makes the decision boundaries of KNN show problem as "The Decision Boundaries of KNN classifier" graph before. For each region, due to it calculates the Euclidean distance of k number of neighbors. It makes each region is only able to show single color. Choose a high-value k will generate more voters in each prediction to solve this problem [10]. However, in this part, we used the default setting with k =5. Therefore, some points will not able to be located in the correct region. In the graph, one black point is in the yellow area and some yellow points are in the dark area. It caused by the distance of these points is not far enough to generate k-neighbors. It makes them in the wrong location and causes accuracy is 0.96.

The Decision Tree Classifier algorithm shows the best performance in these three algorithms. The accuracy is 1.0 and all points are located in the correct region. This algorithm performed well for solving decision-related problems [11]. At the same time, due to the feature of this dataset is simple. It would not cause an over-fitting problem for the Decision Tree Classifier algorithm. These points make Decision Tree Classifier is a sensible approach for working on this dataset.

Q3.2 Answer¶

The Naive Bayes is a better approach for working with a very large number of data.

The running time of Naive Bayes is :

O(d*c)

d is the query vectors dimension and c is the total class [14].

The running time of Decision tree is :

O(n m logn) [4]

where n is the number of sample, m is the number of features.
